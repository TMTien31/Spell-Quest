2.1Types + Config SchemaFoundation — làm đầu tiên
Thêm vào types.ts
// Effect types
export type EffectId =
  | 'delay'        // gõ sai → chờ Xs trước khi gõ lại
  | 'poison'       // mất HP theo tick
  | 'wound'        // dame nhận tăng thêm sau mỗi hit
  | 'streakBreak'  // xóa streak về 0
  | 'manaDrain'    // trừ mana (placeholder cho Phase 3)
  | 'shieldBreak'  // xóa toàn bộ shield
  | 'healBlock'    // chặn mọi nguồn hồi HP

export type EffectTarget = 'player' | 'boss'
export type EffectTrigger = 'onWrong' | 'onHit' | 'onTick' | 'onEncounterStart' | 'immediate'

export interface ActiveEffect {
  id: EffectId
  remainingDuration: number   // số từ còn lại (hoặc giây nếu tick-based)
  params: Record<string, number>  // { damage: 8, interval: 4 } v.v.
  source: 'boss' | 'item'
}

// Boss skill types
export interface BossSkill {
  id: string
  name: string
  triggerAtHit: number        // tung skill sau N lần boss hit thành công
  effectId: EffectId
  params: Record<string, number>
  durationWords: number       // effect kéo dài bao nhiêu từ (0 = instant)
  phase: 1 | 2 | 'both'
}

export interface BossConfig {
  entityId: string
  phase2Threshold: number     // % HP kích hoạt phase 2 (mặc định 0.5)
  phase2Passive: {            // passive boss nhận khi vào phase 2
    type: 'regen' | 'lifesteal' | 'timerRush'
    value: number
  } | null
  phase2PlayerDebuff: {       // debuff player nhận khi boss vào phase 2
    effectId: EffectId
    params: Record<string, number>
    durationWords: number
  } | null
  skills: BossSkill[]
}

2.2Effect Engine — core logicPhức tạp nhất, quan trọng nhất
Tạo src/utils/effectEngine.ts (file mới)
// 3 function chính:

// 1. Apply effect vào state
applyEffect(
  effect: { effectId: EffectId, params, durationWords },
  playerState: PlayerState,
  source: 'boss' | 'item'
): PlayerState

// 2. Tick effects sau mỗi từ (giảm duration, xóa effect hết hạn, apply tick damage)
tickEffects(
  playerState: PlayerState,
  event: 'onCorrect' | 'onWrong' | 'onNewWord'
): PlayerState

// 3. Check instant effects (delay, streakBreak) — trả về có nên block input không
checkInstantEffect(
  effectId: EffectId,
  playerState: PlayerState
): { shouldBlock: boolean, blockDurationMs: number }
Logic từng effect
delay: Khi gõ sai → set flag inputBlockedUntil = Date.now() + params.duration. CombatView check flag này trước khi cho phép gõ.
poison: Mỗi N giây (interval từ params) → trừ params.damage HP. Dùng useEffect + setInterval trong CombatView, đọc activeEffects để biết có poison không.
wound: Khi player nhận damage → damage × (1 + stacks × 0.2). Stack tăng mỗi lần bị hit khi wound active. Max stacks từ config.
streakBreak: Instant — set streak về 0 ngay khi apply.
shieldBreak: Instant — set shield về 0 ngay khi apply.
healBlock: Flag trong PlayerState. Mọi logic +HP check flag này trước khi apply.
manaDrain: Placeholder — trừ mana (mana = 0 hiện tại, Phase 3 mới có ý nghĩa). Log warning nhưng không crash.
Tích hợp vào App.tsx / CombatView.tsx
Thêm activeEffects: [] vào PlayerState khởi tạo
Trong handleCorrectAnswer: gọi tickEffects(state, 'onCorrect') trước khi +shield +streak
Trong handleWrongAnswer: gọi tickEffects(state, 'onWrong'), sau đó check delay effect
Khi encounter kết thúc: xóa toàn bộ activeEffects (effects không carry sang encounter tiếp)
Dependency: Cần 2.1 xong trước.

2.4Phase 2 TriggerNhỏ — hook vào boss HP check
Logic cần thêm vào combat loop
Thêm isPhase2: boolean vào combat state, khởi tạo false
Sau mỗi lần boss nhận hit: check bossHpPercent <= bossConfig.phase2Threshold
Nếu vừa chuyển sang phase 2 (!isPhase2 && shouldBePhase2): set isPhase2 = true
Apply phase2Passive — nếu là regen: thêm vào combat loop mỗi từ đúng boss nhận +regen.value HP lại
Apply phase2PlayerDebuff lên player qua applyEffect()
Hiện visual indicator: border boss zone đổi từ đỏ thường → đỏ sáng hơn, thêm text "PHASE 2" nhỏ
Dependency: Cần 2.2 + 2.3 xong trước.
